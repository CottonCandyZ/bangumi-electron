### Detail

1. 现在的 #/command 打开的 window，也就是上一个 feat instructs\2026-01-12-13-10-10-feat-enable-command.md 创建的东西有个问题。
2. 其是一个全屏窗口完成的，但是我们根本不需要全屏窗口，我们只需要一个和这个 command 窗口自身一样大的窗口就可以完成。
3. 于是我们需要知道当前系统window的大小，然后把它定为在屏幕中间偏上一点位置就就行了。
4. 由于这个窗口一定是 electron 已经启动了之后才会出现的，所以我理解我们可以拿到系统 window 的大小。
5. 我印象里对于这个由快捷键呼出的 command tools 有几个特征。

- 它的 overlay 应该被我们干掉了
- 它的位置仍然假设由 webview 自身计算，好像是居中的 dialog + vh + max h 这样的设计。
- 我们需要特殊处理，因为我们的 webView 应该和它本身差不多大。或者说计算方式几乎一致，就是基于当前屏幕的高度算一个高度，有一个跟着变化的百分比固定高度和一个最大的固定高度。总之高度是固定的。
- 同时宽度其实也是固定的，就是这个 Command 本身的宽度，我觉得没啥问题。
